# 🏛️ Интерактивный Музей (interactive-museum-67)

Веб-платформа для цифровизации музейных пространств, автоматизации создания экспозиций и повышения вовлеченности посетителей. Проект разработан в рамках хакатона и представляет собой готовую production-ready систему, объединяющую современный бэкенд на Go, отзывчивый фронтенд на React и оркестрацию через Docker Compose.

---

## 🚀 Основные возможности системы

* **Локальная автономная генерация QR-кодов:** Система полностью независима от сторонних интернет-сервисов. QR-коды генерируются прямо на бэкенде в момент инициализации базы данных или создания экспоната, кодируются в формат Base64 (Data URI) и сохраняются в БД.
* **Мгновенное скачивание QR-кодов:** На странице экспоната музейные работники могут в один клик скачать сгенерированный QR-код в формате .png с автоматическим формированием красивого имени файла (например, QR_Рукомойник.png) для последующей печати и размещения у реальных стендов.
* **Интерактивная мини-игра (Самопроверка):** Каждая страница экспоната содержит интерактивный блок с вопросом на внимательность или знание истории артефакта. Если пользователь отвечает неверно, система мягко направляет его, выводя историческую подсказку (hint) из базы данных.
* **Безупречная мобильная адаптация:** Интерфейс сайта спроектирован по принципу Mobile-First. Фирменные графические элементы (Шадринский Гусь и исторические иллюстрации музея) адаптивно масштабируются с помощью медиа-запросов и CSS-свойств, превращаясь на смартфонах в аккуратные фоновые подложки с регулируемой прозрачностью, не перекрывающие текст.
* **Кроссплатформенная контейнеризация:** Проект упакован в контейнеры и без проблем разворачивается одной командой на Linux (Ubuntu/EndeavourOS), macOS (включая Apple Silicon M1/M2/M3) и Windows (через WSL2).

## 🛠️ Технологический стек

### Бэкенд (Backend)
* Язык программирования: Go (Golang) версии 1.24+ (высокая производительность, легковесные горутины).
* Маршрутизация: Родной высокопроизводительный http.NewServeMux (введенный в Go 1.22).
* База данных: MySQL 8.0 (надежная реляционная СУБД с поддержкой внешних ключей и каскадных удалений).
* Генерация QR: github.com/skip2/go-qrcode (полностью автономная генерация матричных кодов в памяти).

### Фронтенд (Frontend)
* Библиотека интерфейса: React 18+ (компонентный подход, управление состояниями через хуки useState, useEffect).
* Сборщик проекта: Vite (сверхбыстрая сборка, моментальный Hot Module Replacement).
* Веб-сервер для раздачи статики: Nginx (Alpine) (минималистичный образ, настроенный как Reverse Proxy для проксирования API-запросов на Go-сервер во избежание проблем с CORS).

## ⚙️ Быстрый запуск (Инструкция по развертыванию)

Проект полностью автоматизирован. Вам больше не нужно отдельно настраивать базу данных, скачивать зависимости Node.js или компилировать Go-код на хост-машине.

### Шаг 1: Подготовка среды
Убедитесь, что у вас запущен Docker и установлена утилита Docker Compose (в Windows и macOS она встроена в Docker Desktop, на Linux устанавливается пакетом docker-compose-plugin).

* Внимание: Если у вас локально запущены сторонние службы MySQL (порт 3306) или сервера на портах 8081 / 3000, остановите их перед запуском.

### Шаг 2: Клонирование и запуск
Откройте терминал в корневой папке проекта (где лежит файл docker-compose.yml) и выполните команду:

# На Linux или macOS / Windows (в PowerShell):
```
docker compose up -d --build
```
### Шаг 3: Проверка статуса
Посмотрите, что все контейнеры успешно запустились и работают:
```
docker compose ps
```
### Доступ к сервисам
После успешного запуска проект доступен по следующим адресам:
* Интерфейс музея (Фронтенд): http://localhost:3000
* Сервер API (Бэкенд): http://localhost:8081
* База данных (MySQL): localhost:3306 (Пользователь: root, Пароль: 123456, База: museum_db).

Для остановки всех контейнеров используйте команду: 
```
docker compose down.
```

## 📡 Спецификация API (Основные эндпоинты)

Все запросы к API с фронтенда проксируются через Nginx и имеют единый префикс /api/.

### Публичный интерфейс
* GET /api/exhibits — Получить полный список экспонатов музея (с кратким описанием, изображениями и встроенными Base64 QR-кодами).
* GET /api/exhibits/{id} — Получить подробные данные конкретного экспоната, включая его полное описание и привязанный интерактивный вопрос.
* POST /api/game/check — Отправить ответ пользователя на проверку.
    * Тело запроса (JSON): {"exhibit_id": 1, "answer": "из глины"}
    * Особенность: Текст ответа автоматически приводится к нижнему регистру перед отправкой, исключая ошибки из-за заглавных букв.
    * Ответ (JSON): {"is_correct": true, "message": "Правильно!", "hint": ""} (или is_correct: false с выводом исторической подсказки).

### Панель администратора (CMS)
Для выполнения данных запросов (например, через Postman) в заголовках необходимо передавать токен авторизации: Authorization: Bearer super-museum-secret.
* POST /api/exhibits — Добавление нового экспоната (автоматически генерирует для него локальный QR-код).
* PUT /api/exhibits/{id} — Изменение информации об экспонате.
* DELETE /api/exhibits/{id} — Каскадное удаление экспоната и связанного с ним вопроса из базы данных.
