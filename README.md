# Museum API — Backend (Go)

Бэкенд-часть проекта интерактивного музея «Моя милая малая Родина». Построена на языке **Go** с использованием базы данных **MySQL**.

## 🛠 Технологический стек
* **Язык:** Go 1.22+ (используется новый стандартный роутер `ServeMux`)
* **База данных:** MySQL 8.0 (в Docker-контейнере)
* **Конфигурация:** `.env` (библиотека `godotenv`)
* **QR-коды:** Автоматическая генерация через внешний API (qrserver.com)

---

## 🚀 Быстрый запуск

### 1. Подготовка базы данных (Docker)
Запустите контейнер с MySQL:
`docker run --name museum-db -e MYSQL_ROOT_PASSWORD=123456 -e MYSQL_DATABASE=museum_db -p 3306:3306 -d mysql:latest`

### 2. Настройка окружения
Создайте файл `.env` в корне проекта:
```
PORT=8081
DB_DSN=root:123456@tcp(127.0.0.1:3306)/museum_db
ADMIN_SECRET=super-museum-secret
FRONTEND_URL=http://localhost:3000
```
### 3. Запуск сервера
```
`go get github.com/joho/godotenv`
`go get github.com/go-sql-driver/mysql`
`go run cmd/app/main.go`
```
Сервер будет доступен по адресу: `http://localhost:8081`

## 📑 API Документация (для фронтенда)

### Общая информация
* **Base URL:** `http://localhost:8081/api`
* **CORS:** Включен для всех источников (`*`).
* **Авторизация (для CMS):** Требуется заголовок `Authorization: Bearer <ADMIN_SECRET>`.

### 🏛 Экспонаты
* `GET /exhibits` — Получить список всех экспонатов (краткое описание).
* `GET /exhibits/{id}` — Детальная информация об экспонате + вопрос мини-игры (если есть).
* `POST /exhibits` — **(Защищено)** Создание экспоната + автогенерация QR.
* `PUT /exhibits/{id}` — **(Защищено)** Обновление данных.
* `DELETE /exhibits/{id}` — **(Защищено)** Удаление (автоматически удаляет и связанные вопросы).

### 🎮 Мини-игра
* `POST /game/check` — Проверка ответа пользователя.
    * **Request Body:** `{"exhibit_id": 1, "answer": "текст"}`
    * **Response:** `{"is_correct": bool, "points": int, "message": "...", "hint": "..."}` (подсказка приходит только при ошибке).

## 🔐 Безопасность
Все эндпоинты на изменение данных (POST/PUT/DELETE) требуют Bearer-токен. В разработке используйте:
`Authorization: Bearer super-museum-secret`

---

## 🧩 Особенности
1. **Авто-миграции:** При запуске сервер сам создаст таблицы `exhibits`, `quiz_questions` и `team_members`.
2. **Seed-данные:** Если база пуста, сервер автоматически создаст тестовый экспонат («Шигирский идол») и вопрос к нему.
3. **QR-коды:** При создании экспоната бэкенд генерирует ссылку на страницу фронтенда `FRONTEND_URL/exhibit/{id}` и сохраняет её в базе в виде готового URL изображения QR-кода.
